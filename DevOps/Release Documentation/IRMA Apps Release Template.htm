1.       30 Minutes Before Downtime

a.       Create and confirm releases with appropriate build instances (IRMA Client, Icon, Mammoth).

                                                               i.      Create IRMA v__IrmaVer__ Client release

1.       Ensure version var set correctly for any IRMA Client release.

2.       Deploy target env, but do not complete pre-approvals.

                                                             ii.      Create Icon v__IconVer__ release

1.       Deploy target env

                                                           iii.      Create Mammoth v__MammothVer__ release

1.       Deploy target env

2.       15 Minutes Before Downtime

a.       Hold all Tidal jobs.

b.       Do not approve IRMA Client release until downtime begins there is no pre-deployment phase; it blocks users from accessing the IRMA Client and immediately begins deployment.

c.       Make sure Release BA has coordinated with interfacing teams and they have stopped their jobs.

                                                               i.      FAST

                                                             ii.      Financial Systems

                                                           iii.      Data Warehouse

                                                           iv.      DVO

d.       Complete pre-approvals for Icon and Mammoth.

                                                               i.      This starts pre-deploy tasks to disable web apps and stop interfaces. Release waits for manual intervention after pre-deploy phase.

e.       Wait for each release process to reach manual point.

 

2.    Database Pre-Deployment (DBA)

1.       Proceed only as directed by IRMA Release Manager

2.       Execute DB update:

a.       Script: Mark DB Offline For Maintenance

b.       Query mode: normal

c.       Target:

                                                                           i.      __IrmaDbServer__ (where xx = __IrmaDbUpdateRegionList__)

                                                                         ii.      __IconDbServer__

                                                                       iii.      __MammothDbServer__

d.       Group query: OK

e.       Results/output: status only

3.       Set user-access level for all DBs: restricted_user with rollback immediate

 

Start of Downtime Window

 

3.    IRMA Deployment (IRM, DBA)

1.       (DBA) Restrict ItemCatalog (IRMA) DB (with rollback immediate)

2.       (IRM) Approve IRMA-Client release to continue

3.       (DBA) Execute DB update:

a.       Script: IRMA __IrmaVer__ DB Updates

b.       Query mode: SQLCMD

c.       Target: __IrmaDbServer__.ItemCatalog (regions=__IrmaDbUpdateRegionList__)

d.       Group query: OK

e.       Results/output: tab-delimited to file, saved to DB Results folder

4.       (All) Post-DB-Update Tasks: __IrmaPostDbUpdateTask__

5.       (IRM) Validate IRMA Client VSTS release completed without errors

6.       (IRM) Validate IRMA Client version and file count

7.       (IRM) Check IRMA DB update log (confirm run for all regions, correct target server/env, no errors, etc.)

8.       (IRM) Resume IRMA Client release (it will bring IIS app pools online)

9.       (IRM) Validate any other IRMA components

10.   (DBA) When approved by IRMA Rel Mgr, set user-access level for IRMA DBs: multi_user

11.   (IRMA) Begin smoke testing IRMA Client

12.   (__AdditionalIrmaDeployImplementerRef__) Additional IRMA component deploy:
__AdditionalIrmaComponentDeploySection__

13.   (IRM) Update IRMA Suite version

 

4.    Icon Deployment (IRM, DBA)

1.       (DBA) Execute DB update:

a.       Script: __IconDbScriptLink__

b.       Query mode: SQLCMD

c.       Target: __IconDbServer__.Icon

d.       Group query: N/A

e.       Results/output: tab-delimited to file, saved to DB Results folder

2.       (IRM) Approve Icon release to continue

3.       (IRM) Check Icon DB update log (confirm run for correct target server/env, no errors)

 

5.    Mammoth Deployment (IRM, DBA)

1.       (DBA) Execute DB update:

a.       Script: __MammothDbScriptLink__

b.       Query mode: SQLCMD

c.       Target: __MammothDbTarget__

d.       Group query: N/A

e.       Results/output: tab-delimited to file, saved to DB Results folder

2.       (IRM) Approve Mammoth release to continue

3.       (IRM) __MammothManualDeploy__

4.       (IRM) Check Mammoth DB update log (confirm run for correct target server/env, no errors)

b.       __MammothDbServer__.Mammoth

 

8.    Smoke Testing, Release Validation (All)

1.       (All) Perform smoke testing

2.       (IRM) Verify web apps load (Icon Web, Mammoth Web Support)

3.       (IRM) Get commit-to-release approval from testing/validation teams

 

9.    Post-Deployment (All)

1.       (TIBCO Admin) Start any TIBCO apps that were deployed.

2.       (IRM) Ensure TIBCO applications start without error and connect successfully to configured endpoints.

3.       (IRM) Validate release (logs, components)

4.       (IRM) Resume Icon VSTS release (it will start interfaces)

5.       (IRM) Resume Mammoth VSTS release (it will start interfaces)

6.       (All) Interfacing teams can restart/run their jobs

7.       (IRM) Have DBA run tasks to mark DBs open for business (disable offline-for-maint mode)

8.       (IRM) Release/resume all Tidal jobs

a.       Ensure PeopleSoft jobs run successfully

9.       (IRM) Restart Icon2VimQueueManagement Windows Service (vm-icon-prd1 only)

10.   (IRM) Check interface logs (Icon and Mammoth)

11.   (IRM) Check error emails

12.   (IRM) Check and rerun failed Tidal jobs, if needed

12. Roll Back Steps

1.       IRMA Roll Back

a.       Client

                                                                           i.      __IrmaClientRollbackSteps__

b.       Revert DB updates

                                                                           i.      Script: IRMA __IrmaVer__ ROLL BACK

c.       Change IRMA Suite version back to __IrmaSuitePreviousVersion__.

2.       Icon Roll Back

b.       Revert DB Updates

                                                                           i.      Script: ICON __IconVer__ ROLL BACK

3.       Mammoth Roll Back

a.       Redeploy VSTS release used for previous __TargetEnv__ deployment

b.       Revert DB Updates

                                                                           i.      Script: Mammoth __MammothVer__ ROLL BACK
